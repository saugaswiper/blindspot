# Brief 002 ‚Äî Full-Text Retrieval Stage 2: UI, Upload, and Latency Fix

**Date:** 2026-06-22  
**Stage in pipeline:** 3 (full-text retrieval) ‚Äî close-out pass before opening stage 4  
**Status:** UNBLOCKED ‚Äî no ops/secrets dependency; builds directly on handoff 102

---

## Goal

Close the three open gaps left by handoff 102:

1. **AC2 latency fix** ‚Äî replace the sequential source chain with a concurrent DOI fanout so chain-fallthrough no longer accumulates latency.
2. **AC5 UI surface** ‚Äî add the "Get full text" button with source badge to `include`-verdict study rows in the workbench; right now the API exists but is entirely invisible to users.
3. **AC6 user-upload fallback** ‚Äî show "Upload PDF" when all automated sources return null so no included study is a dead end.
4. **Persistence** ‚Äî save `fulltext_url` / `fulltext_source` / `fulltext_fetched_at` to the Supabase `screening_results` row so the resolved URL survives page refresh.

---

## Why now

Validation 002 confirmed the resolution logic is sound ‚Äî 100% on the OA gold set, provenance on every result, no paywalled URLs. But the feature is operationally invisible: there is no button in the workbench, no upload escape hatch, and no persistence. A user who screens a study to `include` today has no way to get its full text inside the app.

Data extraction (the next pipeline stage) is gated on full text being reliably available *and* surfaced. Building extraction on top of an invisible, ephemeral retrieval layer would create hidden state. Closing this stage cleanly ‚Äî UI, upload, persistence, latency ‚Äî is the minimum viable foundation for stage 4.

The latency failure (F1, validation 002) also leaves the API fragile: any Unpaywall miss triggers a ~3.4 s OpenAlex serial call. Concurrent fanout is a structural fix, not a micro-optimisation.

## Notes for the dev

**Concurrent fanout pattern (AC2 fix):**
```ts
// DOI-keyed sources: race them
const [unpaywall, openalex] = await Promise.allSettled([
  fetchUnpaywall(doi),
  fetchOpenAlex(doi),
]);
const doiResult = [unpaywall, openalex]
  .filter(r => r.status === 'fulfilled' && r.value !== null)
  .map(r => (r as PromiseFulfilledResult<FulltextResult>).value)[0] ?? null;

// PMID-keyed fallbacks only if DOI fanout returned null
if (!doiResult && pmid) { /* sequential Europe PMC ‚Üí PMC */ }
```
This keeps the PMID chain sequential (they are cheap sources, rarely needed, and racing them adds noise), while ensuring the two common DOI paths never serially accumulate.

**Supabase Storage upload:**
Use `supabase.storage.from('fulltext-uploads').upload(path, file)` in the route handler. The bucket must exist ‚Äî create it in the migration or as a one-time manual step. Return `getPublicUrl()` for now.

**`ScreeningDecision` fields** (already typed in `types/index.ts` per brief 001):
`fulltext_url?: string`, `fulltext_source?: FulltextSource`, `fulltext_fetched_at?: string` ‚Äî confirm these exist before adding them again.

**Validation 002 harness re-run:** after the concurrent fanout is in place, re-run `node spec/validation/002-harness.mjs` unchanged. The harness tests real-network p95; if the outlier study now races rather than chains, the p95 should drop below 3 s. Record the new p95 in a brief inline note; the tester will formally re-verify in validation 003.
