# Brief 001 — Full-Text Retrieval (Open-Access PDF Resolution)

**Date:** 2026-06-22  
**Stage in pipeline:** 3 → 4 gate (after screening, before data extraction)  
**Status:** UNBLOCKED — no ops/secrets dependency

---

## Goal

Build the open-access full-text retrieval layer: given a screened-in study (DOI/PMID from
`ScreeningDecision`), resolve its open-access PDF or HTML full-text URL via a ranked
source chain, store provenance, and surface a "Get full text" action in the screening
workbench for included studies.

---

## Why now

Screening is shipped (handoff 083+). Studies that receive an `include` verdict — or a
`human_decision: "include"` override — currently have no next step in the pipeline. The
roadmap's stated "next milestone" is data extraction (`lib/` doesn't yet contain it), and
data extraction is gated on having full text. This is the missing link.

**None of the required retrieval sources require new secrets:**
- **Unpaywall** (`api.unpaywall.org/v2/{doi}?email=`) — free, DOI-keyed, no API key
- **OpenAlex** `open_access.oa_url` field — already available on the works endpoint when
  OPENALEX_API_KEY is functional, but gracefully skippable if it isn't (CRIT-1 is not
  a blocker here)
- **Europe PMC** OA endpoint — open, PMID/DOI-keyed; same source we already call in
  `lib/europepmc.ts`
- **PubMed Central** `/oai/` and direct PMC PDF paths — open, PMID-keyed

CRIT-1 (OpenAlex API key 401, validation 001 §F1) does not block this brief: Unpaywall +
Europe PMC OA + PMC are enough to deliver a useful resolution chain today.

## Scope

### In
- Open-access URL resolution: Unpaywall (primary), OpenAlex `open_access.oa_url`
  (secondary, gated on key availability), Europe PMC full-text API (tertiary), PubMed
  Central `/oai/` (quaternary)
- User-upload fallback: accept a PDF drag/drop and store `source: "user_upload"` — so
  included studies with no OA version are never a dead end
- Provenance on every result: `source`, `oa_status` (`gold`, `green`, `hybrid`, `bronze`,
  `closed`), `content_type` (`pdf` vs `html`)
- "Get full text" UI action in the screening workbench, scoped to `include`-verdict studies
- Persist resolved URL + source to the Supabase `screening_results` row alongside the
  `ScreeningDecision`

### Out
- PDF parsing, text extraction, or any LLM pipeline over content — that is data extraction
  (next brief)
- Paywalled URLs — the route must never return a URL requiring a subscription
- Full-text screening (using full text to re-evaluate inclusion) — a distinct future stage
- Bulk background pre-fetch — on-demand per study first; background jobs later
- Retraction-watch cross-check at retrieval time — already handled in `lib/retractions.ts`
  at search time; not re-duplicated here

## Acceptance criteria — beat, don't match

Otto-SR auto-fetches open-access PDFs. The bar is: match that capability, then beat it on
**provenance transparency** (Otto-SR is a black box on which source returned what).

| # | Criterion | Measurable bar |
|---|-----------|----------------|
| AC1 | **OA resolution rate** | ≥70% of a held-out test set of 20 known-OA studies (PubMed Central gold set, DOIs verified against Unpaywall) resolve to a non-null URL within the chain |
| AC2 | **Latency** | p95 resolution time ≤ 3 s per study (sequential chain, single study, measured via unit test with real network call or recorded fixture) |
| AC3 | **Provenance on every result** | `fulltext_source` is non-null on every successful resolution; failed resolutions carry a reason code from `{ no_doi, no_pmid, paywalled, all_sources_failed, source_error }` |
| AC4 | **No paywalled URLs** | OA status (`oa_status !== "closed"`) is verified before returning any URL; a unit test asserts this gate exists in the resolution logic |
| AC5 | **Source badge visible in UI** | Resolved studies in the workbench show a labeled source chip (e.g. "via Unpaywall") — not just a link, not just logged |
| AC6 | **User-upload dead-end prevention** | If all automated sources return null, the UI shows "Upload PDF" rather than a disabled state or nothing |
| AC7 | **No regression on screening** | `npx tsc --noEmit --skipLibCheck` and `npx vitest run` pass with zero new failures after the changes |

The **beat** is AC3 + AC5: Blindspot shows exactly which source returned the PDF and its
OA status. A reviewer audit trail that Otto-SR cannot match.

## Notes for the dev

- `ScreeningDecision.doi` and `.pmid` (both `string | undefined`, `types/index.ts:317–318`)
  are the natural input handles — no new upstream plumbing needed.
- The Unpaywall API contract: `GET https://api.unpaywall.org/v2/{doi}?email={your_email}` →
  JSON with `best_oa_location.url_for_pdf` and `oa_status`. No key; use a project email.
- Europe PMC full-text OA: `GET https://www.ebi.ac.uk/europepmc/webservices/rest/{pmid}/fullTextXML`
  (returns XML for PMC-deposited articles). The existing `lib/europepmc.ts` already imports
  the base URL constant — extend there or keep retrieval in the new `lib/fulltext.ts`.
- Keep the source chain pure (no side effects, easy to unit test): `resolveFulltext(doi?, pmid?)`
  returns a plain object; the API route handles caching and DB persistence.
